

跳转原理：

1. 在主应用中配置路由前缀，用于区分各个子应用的路径泳道
2. 子应用的router中默认加上路由前缀


主应用配置
``` javascript
  {

    name: 'reactApp', // 子应用名称
    entry: '//localhost:3000', // 这个可以根据环境变量动态生成
    container: '#container', // 挂载到容器
    activeRule: '/app-react', // 匹配规则前缀， 不需要在真实目录中存在

  },
```

子应用配置

``` javascript

const router = createRouter({

  history: createWebHashHistory(window.__POWERED_BY_QIANKUN__ ? '#/kbj-center/' : '/'),

  routes: publicRoutes

})

```


动态路由

``` javascript

const publicRoutes = [

  { path: '/vue-dgb-admin', key: 'vue-dgb-admin', title: '巡店后台' },

  { path: '/kbj-admin', key: 'kbj-admin', title: '聚合系统' },

  { path: '/kbj-center', key: 'kbj-center', title: '中台系统' }

]

```

`到时候要把每个子节点下的路由补齐`




### 路由处理流程：

1. **主应用URL**：`https://example.com/kbj-center/home`
    
2. **Qiankun识别**：检测到路径匹配 `/kbj-center`，激活子应用
    
3. **子应用加载**：从配置的 entry（如 `https://cdn.example.com/kbj-center-app/`）加载资源
    
4. **子应用内部路由**：使用基础路径 `#/kbj-center/`
    
5. **实际效果**：看起来像是在 `kbj-center` 目录下，但实际上：
    
    - 主应用处理顶层路由 `/kbj-center/*`
        
    - 子应用处理哈希路由 `#/kbj-center/*`
        
    - 服务器只需要提供静态资源，不需要真实目录
